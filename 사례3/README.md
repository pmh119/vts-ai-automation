# [사례 3] VTS 관제일보 자동결산 및 이중 입력 자동화 시스템 (VTS PORT-MIS Auto-Reporter)

해양수산부 PORT-MIS 시스템과 사내 현장업무관리포털(FisdOBS) 간의 복잡한 관제 통계 데이터 수집, 가공, 엑셀 기입, 그리고 이중 수동 입력을 원클릭으로 일괄 자동화해 주는 지능형 업무 자동화(RPA) 시스템입니다.

---

## 1. 개발 배경 및 필요성 (AS-IS)
* **다중 시스템 조회의 번거로움:** 매일 아침 또는 일지 작성 시 관제사들은 해수부 PORT-MIS 포털에 접속하여 당일의 선박 입출항, 이동, 투묘, 이안 실적 데이터를 개별적으로 각각 수동 조회해야 했습니다.
* **엑셀 수동 가공 및 입력:** 수집한 로우(Raw) 데이터를 기반으로 섹터별 통계를 복잡하게 직접 수식 계산하여 내부 엑셀 장부인 **'평택VTS 운영실적.xls'** 및 **'평택VTS 일일통계 계산표.xlsx'**에 매일 직접 기입했습니다.
* **행정포털 이중 입력 유발:** 엑셀 기입을 마친 후, 다시 해양경찰 내부망인 **'현장업무관리포털'**에 접속하여 수집한 결과값을 수동으로 이중 기입(타이핑)해야 하는 비효율이 극에 달했습니다.
* **휴먼 에러 위험 상존:** 복잡한 창 전환과 반복적인 복사/붙여넣기, 수동 데이터 입력 과정에서 데이터 왜곡, 기입 누락, 오타 등 통계 신뢰도를 저해하는 휴먼 에러 발생 우려가 컸습니다.

---

## 2. 해결 방안 및 주요 기능 (TO-BE)
* **임베디드 웹 코어 기반 크롤러:** 셀레늄(Selenium) 등 별도의 크롬 웹드라이버(Chromedriver) 버전 관리가 필요 없는 **Microsoft Edge WebView2 엔진**을 프로그램 내부에 직접 결합했습니다. 보안성 및 폐쇄망이 강조되는 관제실 PC 환경에서 단 하나의 파워쉘 스크립트 실행만으로 안정적으로 웹 자동화를 보장합니다.
* **자동 엑셀 가공 및 기입 (COM Object 연동):** 크롤링된 실시간 포트미스 데이터와 누적 교신량을 바탕으로 복잡한 집계 수식을 자동 연산하고, 로컬 엑셀 파일 지정 셀(`Cell`) 위치에 정확하게 자동 바인딩하여 저장합니다.
* **현장업무포털 원격 연계 기입:** 로그인 세션을 안전하게 유지하여 내부 행정망인 현장업무포털(FisdOBS)에 자동 로그인하고, 엑셀에 입력된 최신 집계 결과를 웹 폼 인터페이스에 알아서 100% 매칭 입력 및 전송합니다.
* **직관적인 모니터링 GUI 및 터미널:** 윈도우 폼 기반의 단일화된 조작 패널(GUI)과 실시간 프로세스 진행 상황을 모니터링할 수 있는 가상 터미널(`vts-terminal`) 콘솔 뷰를 제공하여 작업 신뢰도를 대폭 높였습니다.

## 3. 구성 파일 정보
* **`portmis.ps1`:** WebView2 엔진 로드, 윈도우 GUI Form 렌더링, Excel COM 연동 및 웹 크롤링/오토필(Auto-fill) 전 과정을 총괄하는 메인 RPA 제어 프로그램입니다.
* **`평택VTS 일일통계 계산표.xlsx`:** 당일 실시간 1, 2, 3섹터별 상세 입출항 데이터가 자동으로 취합·연산되는 템플릿 스프레드시트입니다.
* **`평택VTS 운영실적.xls`:** VTS 센터 전체의 월간/연간 실적이 자동으로 기록되어 축적되는 공식 실적 관리 대장입니다.

## 4. 구현 기술 (Tech Stack)
* **OS Environment:** Windows 10/11 (VTS 내부 행정망 PC)
* **Language/Script:** Windows PowerShell v5.1+
* **GUI & Web Framework:** 
  * .NET WinForms (`System.Windows.Forms`, `System.Drawing`)
  * Microsoft WebView2 WinForms SDK (`Microsoft.Web.WebView2.WinForms.dll`)
* **Office Automation:** Microsoft Excel COM Interop Object Model (`Excel.Application`)
* **Database & Security:** Windows Credential Manager 연계 안전 계정 저장 및 XML 기반 로컬 세션 캐싱

## 5. 설치 및 사용 방법

### 사전 필수 설치 요소
스크립트의 안정적인 동작을 위해 아래 WebView2 DLL 파일들이 해당 경로에 위치해야 합니다.
* **DLL 경로:** `C:\WebView2\Microsoft.Web.WebView2.Core.dll` 및 `Microsoft.Web.WebView2.WinForms.dll`

### 실행 방법
1. **`portmis.ps1`** 파일을 마우스 우클릭하여 `PowerShell에서 실행`을 누릅니다.
2. 실행 시 나타나는 통합 화면 GUI에서 **PORT-MIS 계정 정보**와 **현장업무포털 계정 정보**를 최초 1회 입력합니다. (정보 저장 체크 시 자동 암호화 저장됨)
3. 엑셀 실적 템플릿 파일들의 로컬 경로를 선택합니다.
4. **"엑셀(운영실적) 및 현장업무포털 실적 입력 실행"** 대형 버튼을 클릭합니다.
5. 하단 터미널 화면에 실시간으로 데이터 취합, 엑셀 기입, 웹 포털 전송 단계가 상세히 표출되며 약 **1분 내에 모든 이중 입력 작업이 종결**됩니다.

## 6. 기대 효과 및 도입 성과
1. **행정 처리 시간 95% 단축:** 기존 매일 아침 수동 조회 및 이중 입력으로 인해 소요되던 **10~20분 이상의 복잡한 절차가 단 1분으로 대폭 축소**되었습니다.
2. **입력 오차율 0% 달성:** 사람이 눈으로 보고 타이핑하던 방식에서 엑셀 개체 데이터 다이렉트 바인딩 방식으로 변환되어 **휴먼 에러를 완벽하게 배제**했습니다.
3. **업무 연속성 극대화:** 복잡한 드라이버 설치와 버전 불일치 문제가 잦은 일반 RPA 솔루션과 달리, Edge 코어를 직접 이식한 초경량 하이브리드 자동화 코드로 **유지보수 비용을 최소화**했습니다.

## 7. 보안 및 운영 환경 제약 사항 (Security & Operational Constraints)

본 시스템은 국가 해양교통안전망 및 해양경찰 내부 행정망에 연결되는 만큼, 다음과 같은 보안 조치와 운영 제약 사항을 준수하여 운용됩니다.

### 1) 내부망 접속 및 특정 IP 제약 사항 (Network & IP Access Control)
* **PORT-MIS 시스템 접속 제한:** PORT-MIS 웹 서비스는 해양수산부에서 공식 승인 및 지정해 준 **VTS 센터 내의 특정 고정 IP 주소를 가진 PC 단말기에서만 로그인 및 실적 조회가 가능**합니다. 외부망이나 미승인 IP 환경에서는 접근 자체가 차단됩니다.
* **현장업무포털(hp.kcg) 접속 제한:** 현장업무관리포털 서비스(`hp.kcg.go.kr`)는 **해양경찰 내부망(VTS 센터 내부 업무 단말기)**에서만 작동하며, 일반 외부 인터넷 환경에서는 도메인 조회 및 접근이 불가능한 강력한 보안 제약을 가집니다.

### 2) 소스 코드 민감 정보 제거 및 더미 처리 (Data Masking)
* **내부망 도메인 마스킹:** 보안 유지를 위해 `portmis.ps1` 소스 코드 내 **23번째 및 24번째 줄**에 선언되어 있던 해경 내부망 실제 URL 도메인(`hp.kcg.go.kr`)을 안전한 더미 도메인인 **`hp.kcg.internal`**로 원격 마스킹(더미) 처리 완료하였습니다.
  * **23번 줄:** `$FIELD_LOGIN_URL   = "http://hp.kcg.internal/FisdOBS/index.jsp"`
  * **24번 줄:** `$FIELD_DIARY_URL   = "http://hp.kcg.internal/FisdOBS/vt/vtsWorkDiary.do?progrm_id=OBS0501"`
* **엑셀 템플릿 민감 데이터 소거:** 동봉된 실적 대장 파일(`평택VTS 운영실적.xls` 및 `평택VTS 일일통계 계산표.xlsx`) 내 실제 VTS 센터 운영 실적 및 선박 통계 데이터는 혹시 모를 기밀 유출 방지를 위해 **모두 깨끗하게 소거되었으며, 시스템 오토 바인딩을 위한 테이블 서식(틀)만 온전하게 남겨둔 상태**로 배포하였습니다.

### 3) 향후 암호화 보완 계획 (Future Security Roadmaps)
* **자격 증명 암호화 개선:** 현재 계정 정보를 임시 텍스트(로컬 세션)에 보조 저장하는 로직은 내부 VTS 센터망에서의 간이 편의 목적으로 설계된 부분으로, 별도의 정식 보안성 검토를 거치지 않은 버전입니다.
* **보안 가이드라인 준수:** 향후 정부 및 기관의 고도화된 정보보안 가이드라인에 부합하도록, **Windows 자격 증명 관리자 API(Credential Manager)** 또는 **DPAPI(Windows Data Protection API)** 기반의 개인 식별정보 및 자격 증명 양방향 암호화 모듈로 안전하게 수정/개선할 예정입니다.
