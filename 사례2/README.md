# [사례 2] 실시간 디지털 VHF 교신량 가시화 위젯 (VTS VHF Traffic Widget)

VTS 관제사들이 매일 수작업으로 집계해야 했던 디지털 VHF 무선 통신 교신량을 백그라운드에서 실시간으로 자동 산출하고 화면에 상시 표출해 주는 초경량 대시보드 위젯입니다.

---

## 1. 개발 배경 및 필요성 (AS-IS)
* **자정 마감 업무의 피로도:** 관제사들은 일일 마감 시간인 자정이 되면 관제일지 작성을 위해 당일 누적 VHF 교신 건수를 직접 수동 취합해야 했습니다.
* **불편한 수동 필터링 방식:** 디지털 VHF 녹음 시스템(WGC VCCS)이 저장하는 로컬 폴더에 접속하여, 노이즈나 무의미한 신호를 제외하기 위해 **"크기 16KB 이상이고 파일명에 TX(송신)가 포함된 파일"**만 정렬 및 필터링하여 카운팅해야 하는 번거로운 과정이 있었습니다.
* **실시간 정보 부재:** 상시 교신량이 누적되는 현황을 즉시 파악할 수 없어 당일 업무 부하 정도를 정량적으로 가늠하기 어려웠습니다.

---

## 2. 해결 방안 및 주요 기능 (TO-BE)
* **상시 가시화(Topmost Widget):** 관제 화면 구석에 상시 띄워둘 수 있는 콤팩트한 전용 위젯창(Topmost 속성 적용)을 제공하여, 관제에 방해를 주지 않으면서 언제든 오늘의 누적 교신량을 확인할 수 있습니다.
* **실시간 자동 필터링 엔진:** 백그라운드에서 `Get-ChildItem` 및 크기 필터링 로직이 작동하여, 지정된 로컬 폴더를 스캔하고 "16KB 이상 + TX 파일" 조건에 부합하는 파일 개수만 실시간으로 즉시 합산합니다.
* **자동/수동 새로고침 유연성:** 5초 주기로 교신 정보를 자동 갱신(Auto-Refresh)하는 타이머 기능이 내장되어 있으며, 필요 시 수동으로 갱신하는 새로고침 버튼도 함께 지원합니다.
* **역동적 UI 인디케이터:** 자동 새로고침이 활성화 상태일 때 0.5초 간격으로 회전/깜빡이는 텍스트 인디케이터(●)를 버튼 우측에 깜빡이도록 구성하여 위젯이 정상 작동 중임을 직관적으로 보여줍니다.

---

## 3. 구현 기술 (Tech Stack)
* **OS Environment:** Windows (VTS 관제실 VHF 연동 워크스테이션)
* **Language/Script:** Windows PowerShell v5.1+
* **GUI Framework:** .NET Windows Forms (`System.Windows.Forms`)
  * `Form.Topmost = $true`: 위젯이 타 시스템 화면에 의해 가려지지 않도록 최상단 고정
  * `Timer`: 5,000ms 주기(데이터 갱신) 및 500ms 주기(UI 동작 표시용 시각 피드백) 멀티 타이머 구현
* **File System Operations:** `Get-ChildItem` API 기반 대용량 파일 목록 고속 필터링

---

## 4. 소스 코드 구조 및 설명
`교신량 위젯.ps1`은 크게 **데이터 연산부(Event Handler)**와 **사용자 인터페이스(UI Component)**로 구성되어 있습니다.

```powershell
Add-Type -AssemblyName System.Windows.Forms
$p = "C:\Program Files\WGC\VCCS\Client\Record" # 녹음 파일 저장 기본 경로

# --- [데이터 연산부] 실시간 누적 교신량 집계 핸들러 ---
$f = {
  $d0 = Get-Date; $d1 = $d0.AddDays(-1) # 금일 및 전일 날짜 연산
  $res = @($d1, $d0) | % {
    $dt = $_.ToString("yyyy_MM_dd")
    $dt2 = $_.ToString("M월 d일")
    $t = Join-Path $p $dt
    # 해당 날짜 폴더가 존재하면 'TX'가 포함되고 16KB 이상인 파일 개수를 카운팅
    $c = if (Test-Path $t) { 
        (Get-ChildItem $t -filter "*TX*" | Where-Object { $_.Length -ge 16kb }).Count 
    } else { 0 }
    "$dt2 교신량: $($c)건"
  }
  $l.Text = $res -join "`n" # 라벨에 반영
}

# --- [UI 구현부] Windows Forms 창 설정 ---
$obj = New-Object Windows.Forms.Form
$obj.Width=410; $obj.Height=230; $obj.Text="교신량 집계"; $obj.Topmost=1 # 최상단 고정

# 정보 표시 라벨
$l = New-Object Windows.Forms.Label
$l.SetBounds(20,20,410,100); $l.Font="맑은 고딕, 24pt"

# 새로고침 및 타이머 제어용 스위치 버튼 구성
$btn = New-Object Windows.Forms.Button
$btn.SetBounds(20,130,100,30); $btn.Text="새로고침"; $btn.Add_Click($f)

# 5초 주기 백그라운드 갱신 타이머 가동
$tmr = New-Object Windows.Forms.Timer
$tmr.Interval = 5000
$tmr.Add_Tick($f)
$tmr.Start()
```

## 5. 설치 및 사용 방법

### 사전 준비
녹음 파일이 저장되는 기본 경로(`$p`)가 실제 VCCS(통합교신시스템) 녹음 파일 저장 위치와 일치하는지 확인하고, 필요 시 스크립트 내부 경로를 수정해 줍니다.

### 실행 방법
1. **`교신량 위젯.ps1`** 파일을 마우스 우클릭하여 `PowerShell에서 실행`을 클릭합니다.
2. 화면 우측 하단에 오늘의 교신량이 나타나며, 기본적으로 **"자동 새로고침: ON"** 상태로 가동됩니다.
3. 타이머가 작동할 때마다 버튼 끝에 원형 인디케이터(●)가 깜빡이며 작동 중임을 표시합니다.

## 6. 기대 효과 및 도입 성과
1. **업무 시간 극적 단축:** 매일 자정 수작업으로 녹음 폴더를 열고 16KB 이상 TX 파일을 세어가며 카운팅하던 시간이 **완벽하게 제로(0초)화**되었습니다.
2. **휴먼 에러 차단:** 복잡한 용량 정렬 과정 중 발생할 수 있는 누락이나 카운팅 오차를 완전히 없애고 실무 통계의 신뢰성을 확보했습니다.
3. **관제 집중력 강화:** 자정 시간대 행정 작업 부담이 대폭 줄어들어 피로 누적을 방지하고 해상 통제 및 야간 취약 시간대 안전 모니터링에만 전념할 수 있게 되었습니다.

## 7. 보안 및 운영 환경 안내 (Security & Operating Environment)

### 1) 완전 폐쇄망 및 로컬 시스템 운용 (Closed Loop Local Scan)
* 본 위젯은 외부 인터넷이나 네트워크 전송 없이, **VTS 센터 VHF 장비 단말기의 로컬 저장 경로(`C:\Program Files\WGC\VCCS\Client\Record`)에 저장되는 순수 오디오 파일 수 정보만을 분석**합니다.
* 내부망이나 인터넷망에 전혀 위해를 가하지 않으며, 오프라인(폐쇄망) 환경에서 독립적으로 안전하게 가동되므로 외부 유입 위험 및 무단 데이터 송출 위험성이 원천 배제되어 있습니다.

### 2) 제조사/업체 보안 무관성 (Vendor Security Autonomy)
* 녹음 파일의 음성 내용을 복호화하거나 조작하지 않고, 파일 탐색기 수준의 파일 명칭 규칙 및 용량 필터링(TX 포함 + 16KB 이상)만을 수행하는 읽기 전용(Read-only) 프로세스입니다.
* 전용 통신 벤더사(WGC)의 핵심 시스템 알고리즘에 어떠한 위해도 가하지 않으므로, **유지보수 업체 보안 규정 및 솔루션과의 보안성 시비에서 완전히 자유로운 특징**을 지닙니다.
