ℹ️ RPA 구동을 위한 WebView2 라이브러리 설치 및 수집 가이드

🔒 내부망(폐쇄망) 및 외부망 환경별 오프라인 라이브러리 구성 가이드

PowerShell RPA 자동화 스크립트 실행 시 WinForms GUI와 Chromium 브라우저를 연동하기 위해서는 WebView2 핵심 동적 라이브러리(DLL) 모듈이 로컬 특정 경로에 정확하게 탑재되어 있어야 합니다. 환경에 맞추어 라이브러리를 안전하게 수집하고 배치해 주세요.

  • ① WebView2 공식 런타임(엔진) 설치: 아래 Microsoft 공식 다운로드 페이지에서 Evergreen Bootstrapper 또는 Standalone Installer를 다운로드하여 설치하시면 자동으로 시스템에 렌더링 엔진이 갖추어집니다.
    ⚠️ 중요 알림: 이 공식 설치 프로그램은 '순수 브라우저 엔진'을 윈도우에 설치할 뿐이므로, 스크립트 실행에 필요한 3개의 연동 DLL 파일은 포함되어 있지 않습니다! (정상 동작)
    🔗 Microsoft 공식 WebView2 다운로드 페이지 이동 ↗
    ※ [폐쇄망 주의] 오프라인 윈도우 환경인 경우, 외부 PC에서 Standalone 독립 실행형 설치 파일(MicrosoftEdgeWebview2Setup.exe)을 먼저 내려받아 보안 반입 절차를 거쳐 폐쇄망 PC에 직접 설치해 주셔야 합니다.
  • ② 라이브러리 및 DLL 파일 배치 (깃허브 저장소 동봉): 스크립트 실행에 필요한 3개의 연동 DLL 파일은 이미 사용자님의 깃허브 저장소 `vts-ai-automation` 내 `사례3/` 폴더에 동봉되어 공유되고 있습니다!
    컴퓨터에 C:\webview2 폴더를 생성하신 후, 저장소에서 가져온 DLL 파일들을 아래 트리 구조에 맞게 그대로 복사해 배치해 주세요.
    📂 깃허브 저장소 [사례3] 폴더로 바로가기 (DLL 파일 다운로드) ↗ 
    C:\
└── webview2\
    ├── Microsoft.Web.WebView2.Core.dll       (저장소 [사례3/]에 동봉된 파일 복사)
    ├── Microsoft.Web.WebView2.WinForms.dll   (저장소 [사례3/]에 동봉된 파일 복사)
    └── runtimes\
        └── win-x64\
            └── native\
                └── WebView2Loader.dll        (저장소 [사례3/runtimes/win-x64/native/]에 동봉된 파일 복사)
  • 💡 [폐쇄망 실용 팁] 로컬 검색 'Everything'을 통한 수집 방법
    만약 폐쇄망 환경에서 깃허브 접속이 차단되어 파일 반입이 번거롭다면, PC 내의 타 소프트웨어(예: Microsoft Edge 브라우저 등) 설치 경로에서 이미 활성화된 DLL들을 가져올 수 있습니다. 초고속 로컬 검색 프로그램인 Everything(혹은 윈도우 기본 탐색기)을 통해 아래 3개 파일을 검색하여 한곳으로 복사해 수집하셔도 정상 구동이 가능합니다.
    • Microsoft.Web.WebView2.Core.dll (RPA 핵심 컨트롤러 API)
    • Microsoft.Web.WebView2.WinForms.dll (윈폼 GUI 렌더링 헬퍼)
    • WebView2Loader.dll (Chromium 엔진 로드용 네이티브 핵심 모듈)
  • 💻 아키텍처(32비트/64비트) 및 타 운영체제(OS) 사용자용 안내
    • 64비트(x64) 전용 라이브러리: 현재 저장소에 동봉된 연동 DLL 파일들은 현대 해상교통관제(VTS) 표준 환경에 맞추어 윈도우 64비트(win-x64) 아키텍처용으로 구성되어 있습니다. 만약 32비트(x86) 윈도우 운영체제를 사용 중이신 경우, 스크립트 실행 시 라이브러리 형식 불일치 에러(BadImageFormatException)가 발생할 수 있습니다. 이 경우 Microsoft 공식 NuGet 등을 통해 x86용 WebView2Loader.dll 파일을 다운로드하여 교체해 주셔야 정상 구동됩니다.
    • 윈도우(Windows) 이외의 운영체제: 본 PORT-MIS 자동 보고 스크립트는 폐쇄망 내 오버레이 시각화 구동을 위해 .NET Windows Forms 아키텍처와 Windows native DLL 기술에 의존하고 있습니다. 따라서 macOS, Linux 등 윈도우 이외의 운영체제 환경에서는 스크립트를 통한 위젯/RPA 직접 구동이 불가능하므로, 본 포트폴리오의 실시간 브라우저 기반 작동 예시나 동영상을 참고해 주시기 바랍니다.
