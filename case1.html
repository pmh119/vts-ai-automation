1 프로젝트 핵심 요약
🎯 개발 목적 VTS 시스템 경고창이 다른 화면에 가려지는 현상을 해결하기 위함
기존에는 경고창이 가려지지 않도록 창을 축소하고
추가 창 생성 후 나머지 공간을 채워서 활용하였음
⚙️ 주요 기능 5초 대기 후 선택되어있는 창을 최상단 고정하도록 설정 변경
🛠️ 기술 스택 PowerShell v5.1 · Win32 API (user32.dll) · .NET Framework
PowerShell user32.dll SetWindowPos
✅ 기대 효과 추가로 생성하던 창 제거로 시스템 가용성 확보.
경고창의 자유로운 배치로 시야 확장.
2 도입 전 / 후 비교
⚠️ Before · 기존 방식
  • 경고창 상시 노출을 위해 VOC창과 겹치지 않게 조정
  • 레이다 화면이 좁아져 관제 가시 구역 손실
  • 최대한 자투리 화면 활용을 위해 추가 창 생성
  • 인력손실(창 미세조정), 자원손실(추가 창 생성으로 시스템 과부하)
인력 및 자원손실 발생
✅ After · 자동화 후
  • 스크립트 실행 → 5초 내 창 클릭 → 즉시 최상단 고정
  • 별도 소프트웨어 설치 없이 OS 내장 API만 사용
  • 한번 고정하면 재시작 전까지 영구 유지
  • 레이다 화면 전체를 관제용으로 활용(경고창은 육지 등 미사용 공간에 배치)
편리함 + 시스템 여유 확보

창고정.ps1
⬇ 다운로드 
Add-Type @"
using System;
using System.Runtime.InteropServices;

public class W {
    [DllImport("user32.dll")]
    public static extern bool SetWindowPos(IntPtr h, IntPtr a, int x, int y, int c, int d, uint f);

    [DllImport("user32.dll")]
    public static extern IntPtr GetForegroundWindow();
}
"@

sleep 5

[W]::SetWindowPos([W]::GetForegroundWindow(), -1, 0, 0, 0, 0, 3)
